# Story 015 — Exercise Suggestions Block & Strength Workout Logging Shortcuts

| Field | Value |
|---|---|
| **Story number** | 015 |
| **Type** | Feature |
| **Module** | Dashboard + Log |
| **Layer(s) affected** | UI only (Web — Dashboard.jsx + Strength.jsx) |

---

## User Story

> As an **Athlete**, I want to see today's prescribed strength exercises on the Dashboard and log them quickly without re-entering data, so that I close the recommendation loop and the system learns from what I actually did.

---

## Context — Current State

The Dashboard's `RecommendationCard` renders a freeform `rec.exercises` list. The Garmin API does not provide individual exercise data for strength workouts — strength session details are captured exclusively via manual entry on the Log page (`Strength.jsx`). There is no connection between what the Dashboard prescribes and what the Log page pre-fills: after a gym session, athletes re-enter every exercise from scratch. For athletes whose primary goal is hypertrophy or strength, the recommendation engine outputs a gym session as the primary recommendation — the exercise list is the most important content on their Dashboard, not a secondary section below an endurance block.

---

## New Behavior

**Exercise block rendering:**

- When `DailyRecommendation.exercise_suggestions` is non-null and non-empty, render an Exercise Suggestions block.
- Each row: exercise name, prescribed sets × reps, weight guidance (from `ex.weight_str`, if available), progression flag if the exercise represents a step up from last session.
- When `exercise_suggestions` is null or empty, the block is not rendered. No empty state or placeholder.

**Placement by goal:**

- `goal = 'hypertrophy'` or `goal = 'strength'`: Exercise block renders immediately below the Prescription Card (Story 012), before the Readiness Widget — same visual weight as the card. Gym is the primary recommendation; the exercise list is what the athlete came to the Dashboard to see.
- `sport` is an endurance sport and `exercise_suggestions` is non-null (strength supplement): block renders below the Running Prescription section, labelled **"Strength supplement — if time allows"**. Styling is secondary (smaller heading, reduced visual prominence).

**Post-workout logging shortcuts:**

Three CTAs below the exercise list, navigating to the Log page with a pre-fill payload passed via React Router state:

1. **"Logged it — exactly as prescribed"** — pre-fills a complete session: all prescribed exercises, sets, reps, and weight from `ex.weight_str` (blank if unavailable). Athlete reviews and submits. Zero re-entry for the common case.

2. **"Same exercises, adjusted loads"** — pre-fills exercises and structure (sets, reps); weight fields are left editable. For the common case: followed the plan but went heavier or lighter than prescribed.

3. **"Start from scratch"** — navigates to the Log page with no pre-fill. For sessions that deviated significantly from the prescription.

When the Log page receives a pre-fill payload (shortcuts 1 or 2), it displays a dismissible banner: *"Pre-filled from today's prescription — review before submitting."*

---

## Out of Scope

- Log page internal form changes (pre-fill rendering is Log page scope — this story covers only the Dashboard CTAs and the navigation payload shape)
- Any Garmin API integration for strength data — not available; manual entry is the intended path
- In-dashboard set/rep/weight editing — all editing happens on the Log page
- Mobile implementation (web Dashboard only; mobile parallel to be scoped separately)
- Progressive overload computation or 1RM estimation (Story 005)
- Exercise history or personal records displayed inline on Dashboard

## Acceptance Criteria

- [ ] When `exercise_suggestions` is non-null and non-empty, the block renders with all prescribed exercises (name, sets × reps, weight guidance, progression flag if present)
- [ ] When `exercise_suggestions` is null or empty, no block or placeholder is rendered
- [ ] For `goal = 'hypertrophy'` or `goal = 'strength'`, the Exercise block appears immediately below the Prescription Card with matching visual weight
- [ ] For endurance-primary sessions with a supplement block, the section is labelled "Strength supplement — if time allows" and styled as secondary content
- [ ] All three shortcut CTAs render below the exercise list
- [ ] CTA 1 navigates to Log page with all exercises, sets, reps, and weight pre-filled
- [ ] CTA 2 navigates to Log page with exercises and sets/reps pre-filled; weight fields editable
- [ ] CTA 3 navigates to Log page with no pre-fill
- [ ] Log page shows a dismissible "Pre-filled from today's prescription — review before submitting" banner when arriving via CTA 1 or 2
- [ ] No Garmin strength data is fetched — block renders solely from `DailyRecommendation.exercise_suggestions`
- [ ] Regression: direct Log page access (not via shortcut) works unchanged
- [ ] Regression: endurance-primary athletes do not see a primary-weighted exercise block at the top of the Dashboard
- [ ] When `exercise_suggestions` includes exercises with a follicular set bonus applied (sets count higher than baseline), the increased set count renders correctly without UI truncation
- [ ] Generate Tests

## Technical Notes

`exercise_suggestions` item shape: `{name, sets, reps, weight_str, note, last_done}` — matches current `rec.exercises` structure. Pre-fill payload: pass as React Router `state` on `navigate('/strength', { state: { prefill: exercises, mode: 'exact' | 'template' } })`. Log page reads `location.state?.prefill` on mount and populates the session builder. `goal` is available from `DailyRecommendation` after Story 004. Relevant files: `frontend/src/pages/Dashboard.jsx`, `frontend/src/pages/Strength.jsx`.
