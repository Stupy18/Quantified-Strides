## User Story

> As an **Athlete**, I want running recommendations to include a specific pace range, terrain, and effort cue tailored to my calibrated zone speeds, so that I know exactly how fast to run without guessing.

---

## Context — Current State

Running recommendations from `intelligence/recommend.py` produce a sport and intensity zone but no pace. `user_profile.zone_speeds` does not exist yet (added in Story 001). GAP (grade-adjusted pace) is computed in `intelligence/analytics/terrain_response.py` and `intelligence/analytics/biomechanics.py` but is not used in recommendation output. `workout_metrics.grade_adjusted_pace` is stored but not referenced by the recommendation layer.

---

## New Behavior

**Zone-speed-calibrated pace prescription:**
- `RunningPrescription.pace_range` populated from `user_profile.zone_speeds` for the recommended intensity zone and terrain
- When zone speeds not yet calibrated: `pace_range = None`; narrative notes to run by HR feel instead

**Terrain selection:**
- Road is the default
- Weather precipitation → road
- Training phase: road in Base phase, trail in Build phase (default; overridable by weather)
- Active lower-limb injury with `cross_training_ok = TRUE` → recommendation redirects to cycling, not road running

**GAP flag:**
- `gap_adjusted = TRUE` when session terrain is `trail` AND `biomechanics_baselines` has a trail row
- Trail without baseline: `gap_adjusted = FALSE`; flat road pace ranges used as approximation

**HR/RPE cross-check advisory:**
- `hr_rpe_status = 'decoupled'` → `RunningPrescription.notes` includes: advisory to prioritize Zone 2 and defer threshold sessions
- `hr_rpe_status = None` (< 10 same-sport sessions with both HR and RPE) → no advisory added

---

## Acceptance Criteria

- [ ] When `user_profile.zone_speeds` has a pace range for the recommended zone and terrain, `pace_range` is populated with the formatted string (e.g., "5:45–6:30 /km")
- [ ] When zone speeds are absent for the target zone, `pace_range = None`; no error raised
- [ ] Trail terrain + trail `biomechanics_baselines` row → `gap_adjusted = TRUE`
- [ ] Trail terrain + no trail baseline → `gap_adjusted = FALSE`; road pace ranges used
- [ ] `hr_rpe_status = 'decoupled'` → advisory note present in `RunningPrescription.notes`
- [ ] Fewer than 10 same-sport sessions → `hr_rpe_status = None`; no advisory added
- [ ] Active injury redirecting away from running → `running_prescription = None` in output
- [ ] `RunningPrescription` is `None` when recommended sport is not running
