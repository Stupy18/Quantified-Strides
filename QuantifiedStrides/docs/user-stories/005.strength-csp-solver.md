# Story 005 — Strength CSP Solver

| Field | Value |
|---|---|
| **Story number** | 005 |
| **Type** | Feature |
| **Module** | Recommendation Engine / Strength |
| **Layer(s) affected** | Intelligence layer / API/service |

---

## User Story

> As an **Athlete**, I want the system to automatically select strength exercises that respect my movement pattern fatigue, CNS budget, equipment, and training goal, so that every gym session is optimally loaded without me having to think about what to do.

---

## Context — Current State

The `exercises` table has ~797 exercises but the recommendation engine does not use them for session construction. Exercise suggestions on the Dashboard are noted in `CLAUDE.md` as half-built ("recommendation engine outputs them, Dashboard UI doesn't display them yet"). There is no pattern fatigue tracking, no CNS budget concept, and no `exercise_session_log`. The `strength_sets` table stores completed work but does not drive future recommendations.

---

## New Behavior

**CSP solver — hard constraints (all must be satisfied):**
- Pattern freshness gate: patterns with `pattern_fatigue_residual` above threshold are blocked
- CNS budget cap: sum of `exercises.cns_cost` for selected exercises ≤ session budget (budget derived from TSB + `strength_goal`)
- Equipment filter: only exercises where `exercises.equipment` intersects available equipment
- Injury block: exercises involving `injuries.affected_activities` muscles excluded
- Skill level: `exercises.skill_level` ≤ user's `training_status` tier

**Constraint relaxation fallback:**
- When no solution exists: relax optional soft constraints in order, re-solve
- If still unsolvable: return minimal bodyweight session

**CNS budget modulation:**
- `tsb < −5` → budget reduced 20% from goal baseline
- `strength_goal = 'strength'` + `tsb > 10` → maximum budget, compound exercises eligible
- `strength_goal = 'sport_support'` → minimum budget; `CARRY_BRACE` and `ISOLATION` patterns prioritized

**Pattern fatigue fallback — all patterns gated:**
- Fall back to `ISOLATION` and `CARRY_BRACE` (lowest `fatigue_decay_tau_h`) or return bodyweight session

**1RM cache:**
- Invalidated and recomputed (Epley formula) on every new `strength_sets` insert with reps ≤ 10
- High-rep sets (reps > 10) do not update the cache — Epley unreliable above 10 reps
- Exercise flagged "progression eligible" when last 3 sessions completed all prescribed reps at current load

**Exercise session log:**
- Every CSP-selected exercise written to `exercise_session_log` with `slot_index` and `selected_by='csp_solver'`
- User substitution logged as `selected_by='user_override'` with `constraint_violations_bypassed` populated

**Permanent fixtures:**
- Exercises with `exercises.permanent_fixture = TRUE` appended to every session after solver output, regardless of fatigue or CNS budget

---

## Out of Scope

- UI for the exercise list / session builder (existing `Strength.jsx` is not changed)
- Exercise relationship graph population (`exercise_relationships` table is created but not seeded here)
- Velocity-based training (`velocity_based` column exists but not used in solver v1)

---

## Acceptance Criteria

- [ ] Solver returns an ordered exercise list satisfying all hard constraints
- [ ] No exercise from a pattern with `pattern_fatigue_residual` above threshold appears in output
- [ ] Total `cns_cost` of selected exercises does not exceed the computed session budget
- [ ] When `tsb < −5`, CNS budget is 20% lower than the goal baseline
- [ ] `exercise_session_log` row is written for each selected exercise after session acceptance
- [ ] User override logs `selected_by='user_override'` and populates `constraint_violations_bypassed`
- [ ] New `strength_sets` insert with reps ≤ 10 → `user_1rm_cache` is invalidated and recomputed
- [ ] New `strength_sets` insert with reps > 10 → `user_1rm_cache` is unchanged
- [ ] Exercise with `permanent_fixture = TRUE` always appears last in every session output
- [ ] When no constraints are satisfiable, solver returns a valid bodyweight session (not an error)
- [ ] Pattern cold-start (< 5 ledger entries) → residual approximated from `muscle_freshness`; solver still runs
