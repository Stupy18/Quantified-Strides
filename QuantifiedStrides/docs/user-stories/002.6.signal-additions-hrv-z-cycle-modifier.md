---

## User Story

> As a **developer**, I want `assemble_signals()` to include the raw HRV z-score and cycle modifier decay scale, and compute TRIMP correctly using `biological_sex`, so that the cycle modifier layer and downstream consumers have the signals they depend on.

---

## Context — Current State

Story 002 built `assemble_signals()` with the `hrv_status` categorical string (`'suppressed'`/`'normal'`/`'elevated'`/`'no_data'`) but not the raw z-score float. Story 010 (`apply_cycle_modifiers()`) needs the raw `hrv_z` to apply the late-luteal coaching cue condition (`hrv_z < −1.5 AND tsb > −10`). The cycle modifier decay scale (`cycle_modifier_scale`) — the data-staleness weight from `RECOMMENDATION_PROTOCOL.md §6.4` — is not computed or assembled. Additionally, Story 001 renamed `user_profile.sex → biological_sex` (Story 001.5), but Story 002's TRIMP formula still reads `user_profile.sex`; it must be updated to `biological_sex`.

---

## New Behavior

**`hrv_z: float | None` added to `assemble_signals()` output:**
- The raw z-score computed in `compute_hrv_status()` is now also returned as `hrv_z` in the signals dict
- `hrv_z = None` when `hrv_baseline_sd == 0` or baseline is unavailable (same conditions that produce `hrv_status = 'no_data'`)
- The existing `hrv_status` categorical string is unchanged

**`cycle_modifier_scale: float` added to `assemble_signals()` output:**
- Computed from the number of days since the athlete's last logged cycle start date
- Lookup from `menstrual_cycles` table; most recent `start_date` for the user
- Let `days_since = today − most_recent_start_date` and `median_len = athlete's median cycle length` (last 6 cycles; default 28 if < 3 logged)
- Scale logic:
  - `days_since ≤ (median_len + 2)`: `scale = 1.0` — full modifiers
  - `(median_len + 3) ≤ days_since ≤ (median_len + 9)`: linear decay from 1.0 to 0.0
  - `days_since ≥ (median_len + 10)`: `scale = 0.0` — modifiers disabled
- When `hormonal_contraception = TRUE` or `biological_sex != 'female'`: `scale = 0.0`; no cycle modifiers applied
- When no `menstrual_cycles` row exists at all: `scale = 0.0`

**TRIMP formula reads `biological_sex` (not `sex`):**
- `compute_trimp()` updated to read `user_profile.biological_sex`
- `'female'` → Banister female coefficients (`k=0.86, y=1.67`)
- All other values (`'male'`, `'prefer_not_to_say'`, `NULL`) → male/default coefficients (`k=0.64, y=1.92`)

**`assemble_signals()` dict — two new keys added to the §3.1 signal registry:**

| Key | Type | Source | Nullable |
|---|---|---|---|
| `hrv_z` | `float \| None` | computed in `compute_hrv_status()` | Yes — null when no baseline |
| `cycle_modifier_scale` | `float` | `menstrual_cycles` table | No — defaults to 0.0 |
