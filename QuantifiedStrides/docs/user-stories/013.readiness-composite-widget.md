# Story 013 — Readiness Composite Widget

| Field | Value |
|---|---|
| **Story number** | 013 |
| **Type** | Feature |
| **Module** | Dashboard |
| **Layer(s) affected** | UI only (Web — Dashboard.jsx) |

---

## User Story

> As an **Athlete**, I want to see a single readiness score on the Dashboard that fuses HRV, sleep, TSB, and subjective feel, so that I understand at a glance how ready I am to train today.

---

## Context — Current State

The Dashboard's existing "Readiness Summary" renders individual signals in separate tiles — HRV status, sleep score, TSB value — but there is no fused composite score. `DailyRecommendation.readiness_composite` (a float 0.0–1.0, or null) is produced by the recommendation engine as a weighted fusion of all available readiness signals. It is not currently displayed anywhere on the Dashboard.

---

## New Behavior

**Readiness Composite Widget:**

A compact widget, positioned in the Dashboard between the Prescription Card and any sport-specific prescription blocks, displaying:

- A circular or arc gauge showing `readiness_composite` as a percentage (0–100, derived from `readiness_composite × 100`). The gauge fill colour shifts across a red → amber → green spectrum: 0–39 red, 40–69 amber, 70–100 green.
- A numeric label inside or below the gauge: e.g., "74".
- A plain-language label derived from the score band:
  - 0–39: "Low readiness"
  - 40–69: "Moderate readiness"
  - 70–100: "High readiness"

**Null state:**
- When `readiness_composite` is null (insufficient signal data), the gauge renders in a neutral grey with the label "Readiness unavailable". No score number is shown. This state occurs when the athlete has fewer than 7 days of HRV data or has not synced recently.

**Existing readiness tiles:**
- The existing individual signal tiles (HRV status, sleep score, TSB) remain on the Dashboard unchanged. The composite widget is additive — it does not replace the individual tiles.

---

## Out of Scope

- Breakdown tooltip showing per-signal weights (future enhancement)
- Historical readiness trend chart
- Mobile implementation

---

## Acceptance Criteria

- [ ] Widget renders `readiness_composite × 100` as a rounded integer on a gauge
- [ ] Gauge fill colour: red for 0–39, amber for 40–69, green for 70–100
- [ ] Plain-language band label renders correctly for all three bands
- [ ] Null state: gauge is grey, label reads "Readiness unavailable", no score number shown
- [ ] Widget is positioned between the Prescription Card and any sport-specific blocks
- [ ] Existing HRV, sleep, and TSB tiles are unaffected
- [ ] Regression: Dashboard loads correctly when `readiness_composite` is null

## Technical Notes

Relevant files: `frontend/src/pages/Dashboard.jsx`. `readiness_composite` is a top-level field on `DailyRecommendation`, available in the dashboard API response after Story 004. No backend changes in this story. Gauge can be implemented with SVG arc or a lightweight library — match the existing Dashboard visual style.
