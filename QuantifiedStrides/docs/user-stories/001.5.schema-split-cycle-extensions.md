---

## User Story

> As a **developer**, I want the schema additions required by the split selection system, female athlete cycle modifiers, and open-ended deload tracking in place before Stories 008 and 010 are implemented.

---

## Context — Current State

Story 001 created the baseline schema. Three feature additions designed in Stories 008 and 010 require additional schema that Story 001 did not cover:

1. **Split catalog system** (Story 008 §14.12): `split_catalog` and `split_phase_eligibility` tables do not exist. `training_plan_weeks` and `user_profile` lack split-related columns.
2. **Cycle modifier profile** (Story 010): `user_profile.sex` was added in Story 001 but the field must be renamed to `biological_sex` to match the spec, and two new columns are needed: `menstrual_dysmenorrhea` and `preferred_split`.
3. **Open-ended deload counter** (Story 008 §15.1): `training_plans.open_ended_load_week_counter` does not exist.

---

## New Behavior

No user-visible behavior change. After this story:

**New tables created (2):**

- `split_catalog` — 6 rows seeded; defines each split template with capacity and concurrent-risk metadata
- `split_phase_eligibility` — phase × strength_goal × split eligibility matrix; seeded with all valid combinations

**Columns added / renamed on existing tables:**

- `user_profile`: rename `sex → biological_sex` (no data loss; same TEXT domain); add `menstrual_dysmenorrhea BOOLEAN DEFAULT FALSE`; add `preferred_split TEXT REFERENCES split_catalog(split_key)`
- `training_plan_weeks`: add `split_key TEXT REFERENCES split_catalog(split_key)`, `split_key_overridden BOOLEAN DEFAULT FALSE`, `split_override_acknowledged BOOLEAN DEFAULT FALSE`, `split_override_tsb_cost NUMERIC(5,2)`, `gym_days_override SMALLINT`
- `training_plans`: add `open_ended_load_week_counter SMALLINT DEFAULT 0 NOT NULL`

**Seed data for `split_catalog` (6 rows):**

| split_key | min_days | max_days | concurrent_risk | volume_freq | max_concurrent_aerobic_days |
|---|---|---|---|---|---|
| `full_body_maintenance` | 1 | 2 | low | low | 5 |
| `upper_only_maintenance` | 1 | 2 | low | low | 5 |
| `full_body` | 2 | 3 | medium | medium | 4 |
| `upper_lower` | 3 | 4 | medium | medium | 3 |
| `push_pull_legs` | 3 | 4 | medium_high | high | 2 |
| `push_pull_lower_upper` | 4 | 5 | high | high | 2 |

**Seed data for `split_phase_eligibility`:** Full matrix of (split_key × phase × strength_goal) rows per `RECOMMENDATION_PROTOCOL.md §14.12`. PPL and PPLU rows for `sport_support` in Peak phase have `eligible=FALSE, requires_alert=TRUE, alert_type='split_phase_conflict'`. PPL and PPLU for hybrid goals in Peak have `eligible=TRUE, requires_alert=TRUE, alert_type='split_phase_informational'`. Taper only allows maintenance splits regardless of goal.

## Technical Notes

Relevant spec: `RECOMMENDATION_PROTOCOL.md §14.12` (split catalog DDL), `§6` (biological_sex, menstrual_dysmenorrhea columns), `§15.1` (open_ended_load_week_counter).

Current highest Flyway version after Story 001: implementer should use the next available version number after whatever Story 001 applied. Rename `sex → biological_sex` via `ALTER TABLE user_profile RENAME COLUMN sex TO biological_sex` — simpler and safer than drop + add when the column may have data.

**Migration dependency:** `split_catalog` must be created before `split_phase_eligibility` (FK) and before `user_profile.preferred_split` FK is added.
