# Story 014 — Running Prescription Block

| Field | Value |
|---|---|
| **Story number** | 014 |
| **Type** | Feature |
| **Module** | Dashboard |
| **Layer(s) affected** | UI only (Web — Dashboard.jsx) |

---

## User Story

> As an **Athlete** whose primary sport is running, I want the Dashboard to show my precise pace targets, terrain recommendation, and any grade-adjusted context, so that I can execute today's run session without ambiguity.

---

## Context — Current State

The Dashboard's `RecommendationCard` shows intensity as a freeform string (e.g., "Zone 2, easy effort"). There is no structured pace range, no terrain recommendation, and no grade-adjusted pace (GAP) context. `DailyRecommendation.running_prescription` is a typed sub-object — `{pace_range, terrain, gap_adjusted, notes}` — produced by the running prescription module (Story 006). It is not currently displayed.

---

## New Behavior

**Running Prescription Block** — rendered only when `DailyRecommendation.running_prescription` is non-null:

- **Pace range**: displays `pace_range` as "X:XX – X:XX /km" (or /mile, per user's unit preference). Labelled "Target pace".
- **Terrain**: displays `terrain` as a badge — "Road" or "Trail". If null, terrain badge is omitted.
- **Grade-adjusted**: if `gap_adjusted = true`, a small indicator reads "GAP-adjusted" next to the pace range, signalling that targets account for elevation. If false or null, no indicator.
- **Advisory notes**: renders `running_prescription.notes` as body text. If null, omitted.

The block is positioned below the Readiness Composite Widget (Story 013) and above the Exercise Suggestions Block (Story 015). For endurance-primary sessions, it is the most detail-rich section of the recommendation.

**Null state:**
- When `running_prescription` is null (non-running session, rest day, or insufficient zone calibration data), the block is not rendered. No placeholder shown.

## Out of Scope

- Pace zone calibration or zone speed display (Story 006 — intelligence layer)
- Route suggestions
- Mobile implementation
