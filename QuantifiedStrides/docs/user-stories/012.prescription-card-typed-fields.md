# Story 012 — Today's Prescription Card: Typed Fields

| Field | Value |
|---|---|

| **Type** | Feature |

---

## User Story

> As an **Athlete**, I want the Dashboard to display today's recommendation in a structured, readable card — sport, session type, intensity zone, duration, and any advisory notes — so that I know exactly what to do without having to interpret freeform text.

---

## Context — Current State

The Dashboard's `RecommendationCard` renders a freeform payload: `rec.primary`, `rec.intensity`, `rec.duration`, `rec.gym_rec`, `rec.why`, `rec.narrative`, `rec.avoid`, `rec.notes`. These are untyped strings assembled ad-hoc in the recommendation service. After Story 004 delivers `DailyRecommendation`, the backend returns strongly-typed fields: `sport`, `session_type`, `intensity_zone` (1–5 integer), `duration_min`, `notes`, `confidence`. The frontend card must be rebuilt to consume these fields and replace the old freeform layout.

---

## New Behavior

**Prescription Card — typed layout:**

- **Sport badge**: renders `DailyRecommendation.sport` as a pill badge (e.g., "Running", "Strength", "Cycling").
- **Session type**: renders `session_type` as the card heading (e.g., "Zone 2 Base Run", "Strength — Hypertrophy Block").
- **Intensity zone**: renders `intensity_zone` as a 5-pip visual indicator with the active zone(s) highlighted. If `intensity_zone` is null (rest day), pips are greyed and a "Rest" label is shown.
- **Duration**: renders `duration_min` as "X min". If null, omits the duration line.
- **Advisory notes**: renders `notes` as body text below the main fields. If null, omits the notes section.
- **Confidence indicator**: renders `confidence` as a small label below the card — "High confidence", "Medium confidence", "Low confidence", or "Insufficient data" mapped from 'high'|'medium'|'low'|'no_data'. Low confidence and no_data states show the label in a muted warning colour.

**Rest day state:**
- When `session_type = 'rest'`, the card renders: sport badge hidden, heading "Rest Day", intensity pips greyed, duration and notes rendered as usual.

**Old fields removed:**
- `rec.primary`, `rec.intensity` (string), `rec.gym_rec`, `rec.why`, `rec.narrative`, `rec.avoid` are no longer read by this component. The narrative field moves to the Narrative Card (unchanged). `rec.avoid` is dropped — its content migrates to `alerts` (Story 016).
